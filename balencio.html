Balencio

Native Wealth Simulation & Interface Engine

Platform / Architecture

Android Mobile / MVVM

UI Framework

Jetpack Compose (Material 3)

Target SDK

36 (Android 16 API Sandbox)

Application Purpose & Scope

Balencio is a high-fidelity sandbox simulation application tailored to model modern retail banking, neo-brokerage, and cryptocurrency portfolio interfaces. Designed exclusively for entertainment, configuration testing, and user-experience research, it offers users a secure, localized dashboard to monitor simulated high-net-worth profiles, cash flow transaction sheets, and mock asset holdings without financial risk.

The core objective of the project was to create a flawless financial UI that responds dynamically to user inputs. It translates complex dashboard states into interactive sub-menus for balance modification, decentralized ledger views, and mock stock trends while using strict internal data management to isolate simulations from actual banking APIs or payment networks.

Technical Stack & Implementation

The codebase is written in native Kotlin utilizing a Single-Activity structural model powered by an asynchronous MVVM (Model-View-ViewModel) state production chain. By leveraging declarative architectures, system constraints like hardware flags and permissions run concurrently without blocking UI updates.

Reactive State & Core Lifecycle
  • Asynchronous State Flows: ViewStates are handled via decoupled Kotlin StateFlow pipes, collected safely inside Compose lifecycles using collectAsState().
  • Hardware Window Intercepts: Implements asynchronous SideEffect parameters to dynamically lock or unlock system-level FLAG_KEEP_SCREEN_ON layout configurations based on current user preferences.
  • Edge-to-Edge Compilation: Implements native window insets to allow the UI shell to utilize full screen displays.
Navigation & Graph Topologies
  • Compose Navigation Graph: A centralized navigation host handles seamless switching between functional layers (Splash, Onboarding Intro, Configuration, and the core app).
  • State-Aware Bottom Bar: The Material 3 NavigationBar tracks the backstack destination hierarchy in real time, intelligently hiding itself during onboarding screens and showing itself automatically on main views.
  • State Preservation: Navigation targets use single-top launching parameters and backstack state restoration to prevent memory leakage or view re-initialization.
Hardware & SDK Bridges
  • Modern Permissions Handling: Uses rememberLauncherForActivityResult coupled with contracts to cleanly prompt users for POST_NOTIFICATIONS on Android 13+ devices.
  • Google Play Billing v6 Integration: Uses lifecycle-aware LaunchedEffect blocks to instantiate Google Play Billing wrappers safely for premium in-app purchase processing.
  • Mobile Ads SDK Integration: Ad components are decoupled from UI elements, initializing asynchronously on app launch to manage interstitial caches cleanly.
Data Interceptors
  • Retrofit & GSON Pipelines: Engineered clean API adapters for network queries.
  • Coil Engine: Native background workers fetch and render asset logos without causing layout frame drops.

Functional Core Components

  • Core Dashboard Simulator: A high-fidelity banking overview tracking primary account ledgers, quick actions, and customizable user data profiles.
  • Mock Transaction Engine: Dedicated internal transfer interfaces that accept user-defined parameters to immediately append mock balances.
  • Cryptocurrency & Equity Hub: Replicates decentralised holding portfolios to display custom quantities of tokens like Bitcoin and Ethereum.
  • Dynamic Theme Engine: A customizable Material 3 theme layer that dynamically swaps base parameters, light/dark modes, and primary color schemes on the fly.

Product Interfaces

Meticulously structured minimalist interfaces optimized for readability and fluid user progression.

Architectural Disclaimer: Balencio is built strictly under sandboxed local system rules. It features no actual connection to any production financial clearing houses, credit institutions, or decentralized blockchains. All ledgers, incoming transactions, and asset growths are fully simulated locally.
