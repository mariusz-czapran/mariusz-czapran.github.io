Project Overview & Ownership

Balencio is a high-fidelity simulation application built to mirror contemporary retail banking, neo-brokerage, and crypto-currency asset dashboards. As an entirely self-directed project, I managed the **complete product lifecycle independently**: driving initial product discovery, designing the interface concepts, architecting the native engineering pipeline, coordinating closed beta-testing compliance tracks, and ultimately navigating full production release cycles.

The app is successfully published in production on the **Google Play Store**, serving users worldwide. To accommodate a global user base, the application is fully localized with native multi-language support, adjusting interface strings, regional formatting metrics, and layout elements seamlessly based on device language parameters.

Technical Stack & Implementation

The technical implementation leverages a robust, Single-Activity structural paradigm written in native Kotlin. It enforces a strict asynchronous Model-View-ViewModel (MVVM) state management pipeline to insulate performance-heavy UI components from background operational workflows.

Reactive State & Core Lifecycle
  • Asynchronous State Flows: Application state properties are piped via Kotlin StateFlow networks, handled cleanly via Jetpack Compose's collectAsState() mechanism.
  • Hardware Window Intercepts: Coordinates asynchronous SideEffect parameters to dynamically apply or release system-level FLAG_KEEP_SCREEN_ON layout configurations based on current user preferences.
  • Edge-to-Edge Execution: Integrates native system window insets, allowing layout elements to scale seamlessly behind the navigation bars.
Navigation & Graph Topologies
  • Compose Navigation Graph: A centralized controller handles complex transitions between functional modules (Splash screen, Onboarding flows, User Configuration, and the main application suite).
  • State-Aware Bottom Bar: The Material 3 NavigationBar tracks active destination hierarchies on the fly, safely hiding itself during setup tracks to maximize screen area.
  • State Preservation: Destinations leverage single-top launch paradigms to guarantee swift navigation caching and zero memory regressions.
Hardware & SDK Bridges
  • Modern Permissions Handling: Uses rememberLauncherForActivityResult api contracts to prompt users gracefully for POST_NOTIFICATIONS on Android 13+ devices.
  • Google Play Billing v6 Integration: Uses lifecycle-aware LaunchedEffect blocks to instantiate Google Play Billing wrappers safely for premium feature unlocks.
  • Mobile Ads SDK Integration: Ad layers are entirely decoupled from screen composables, loading asynchronously via lifecycle caches to optimize frame rates.
Localization & Assets
  • Multi-Language Framework: Engineered modular resource translation bundles allowing seamless local presentation for widespread user markets.
  • Retrofit & GSON: Scalable networking layer for retrieving decentralized mock asset tickers.

Functional Core Components

  • Core Dashboard Simulator: A high-fidelity overview managing primary mock asset categories, balances, and quick transaction logs.
  • Mock Transaction Engine: Dedicated internal transfer screens allowing instant generation of simulated fund movements.
  • Cryptocurrency & Equity Hub: Replicates decentralised holdings tracker views, managing localized quantities of major tokens like Bitcoin and Ethereum.
  • Dynamic Styling Controls: A fully customized Material 3 design system layer enabling real-time adjustments of base styles, color configurations, and dark/light modes.
